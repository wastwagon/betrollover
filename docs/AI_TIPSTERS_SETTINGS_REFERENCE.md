## Differences to compare

- **risk_level:** conservative (7), balanced (14), aggressive (4).
- **target_odds:** Most use 1.41–2.0; Under 2.5 Daily and The Gambler use 1.41–2.2; TheAnalyst 1.3–1.5; StatsMachine 1.3–1.7; LateBloomer max 1.6.
- **max_daily_predictions:** 3 for all (same-day fixtures only; no advance/future coupons).
- **bet_types:** Only field that fully defines “market” (1X2, BTTS, Over/Under, Under 2.5, Double Chance, etc.).
- **Extra filters:** HomeHeroes = home_only; TopSixSniper = team_filter top_6.

Source: `backend/src/config/ai-tipsters.config.ts`

## How to check which tipsters are getting coupons

**1. From the database (who has coupons right now)**  
From repo root, with backend env and DB running:

```bash
cd backend && npx ts-node -r tsconfig-paths/register scripts/check-ai-tipster-coupons.ts
```

Shows tipsters with ≥1 coupon and tipsters with 0 (and whether they’re missing in DB or have no `user_id`).  
To limit to recent activity:

```bash
DAYS=30 npx ts-node -r tsconfig-paths/register scripts/check-ai-tipster-coupons.ts
```

**2. Fetch directly from API (no DB)**  
Fetches fixtures + predictions + odds from API-Football only; simulates which tipsters would get coupons (same filters as engine, global `usedFixtureIds`). No database required.

```bash
cd backend && API_SPORTS_KEY=your_key npx ts-node -r tsconfig-paths/register scripts/test-ai-tipsters-from-api.ts
```

Optional: `FIXTURES_LIMIT=30` (default 25) to fetch more fixtures (more API calls).

**3. Dry-run generation (with DB)**  
Uses DB fixtures + API-Football; does not save predictions. Requires DB + API key:

```bash
cd backend && API_SPORTS_KEY=your_key npx ts-node -r tsconfig-paths/register scripts/test-prediction-generation.ts
```

Prints per-tipster coupon counts for the next 7 days and lists tipsters with 0 coupons.
